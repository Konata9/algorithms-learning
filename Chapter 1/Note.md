# 笔记
1. 二分法
  - 条件：有序序列（排序、查找，算法的两大问题）
  - 算法复杂度： O(logn)
  - 100 个数二分法 7 次就能猜到：2^7 = 128 > 100

2. 算法复杂度
  - O(n) 表示算法在最糟糕的情况下运行的时间。不是秒，而是复杂度或者说是时间增长的幅度
    - 线性查找： 1-n 之间
    - 二分查找： 1-logn 之间
  - 常见的算法复杂度与对应的算法：
    1. O(logn) 对数时间 -- 二分法
    2. O(n) 线性时间 -- 线性查找
    3. O(n * logn)  -- 快速排序
    4. O(n^2) -- 选择排序
    5. O(n!) -- 旅行商问题（NP问题）
